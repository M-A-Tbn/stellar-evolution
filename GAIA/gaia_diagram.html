The Two-Layer Rule
Layer 1 — Internal
Agent Folders — Markdown only · not for students
Apollo/session – notes.md · Coeus/session – quiz.md · Metis/session – flashcards.txt · Iris/session – summary.md · Themis/question_bank.md
Layer 2 — Student
STUDY PATH/ — HTML only · Athena is the only producer
notes.html · quiz.html · summary.html · flashcards.html · question_bank.html · transcript – date.html
Rule:  No agent other than Athena ever writes .html. Every agent writes markdown to its own folder. Athena always fully regenerates the HTML from all markdowns — the old HTML is completely replaced on each run. STUDY PATH/ is the only folder the student ever needs to open.
Agents
🔭
Argus
Preprocessor

Gemini CLI invocation that converts PDFs into High-Fidelity markdown with perfect LaTeX rendering and detailed inline figure descriptions. Output is cached — never re-run unless --force is passed.

InStudyMaterials/pdfs/*.pdf
OutStudyMaterials/ocr/*_hifi.md
Cmd/Argus
AltClaude Vision — reads PDF directly; no cache, higher token cost
🌞
Apollo
Note Taker

Exhaustive study note taker. Reads _hifi.md + paired Fortran source. Every concept, formula, algorithm step, and code behaviour is captured and fully explained — nothing omitted, nothing summarised. Always sequential — never dispatched in parallel.

In*_hifi.md + paired Fortran — or raw PDF (Vision fallback)
OutApollo/session – notes.md (up to 3 parts per session)
Cmd/Apollo
RuleOne session at a time. Never reads raw PDFs when _hifi.md is available.
🦉
Athena
HTML Builder

The sole HTML producer. Runs a Python builder script on any agent's markdown files to produce themed, searchable, paged HTML. Always fully regenerates — no partial injection. Never reads raw PDFs or source code.

InProcessed markdown from any agent folder
OutHTML in STUDY PATH/ — one file per agent type, fully rebuilt each run
Cmd/Athena
AutoTriggered automatically after every working agent completes
🧠
Coeus
On-demand

Quiz maker. Mixed format: MCQ, true/false, open-ended. Reads only Apollo's markdown — never raw PDFs or Fortran. Output follows strict format required by coeus_build.py.

InApollo/session – notes.md only
OutCoeus/session – quiz.md
HTMLAthena → STUDY PATH/quiz.html via coeus_build.py
Cmd/Coeus
🃏
Metis
On-demand

Flashcard maker. Produces Anki-compatible tab-separated .txt files (Front\tBack per line). Reads only Apollo's markdown. The .txt file is copied to STUDY PATH/ for direct Anki import.

InApollo/session – notes.md only
OutMetis/session – flashcards.txt (also copied to STUDY PATH/)
HTMLAthena → STUDY PATH/flashcards.html via metis_build.py
Cmd/Metis
🌸
Iris
On-demand / Auto

Summariser. Compact exam-prep summary — ~10–15% the length of the source. Four sections: Key Concepts, Key Equations, Key Algorithms, Watch Out For. Two modes: Standard (student-triggered) or Question-Aware (auto after Themis Phase 1).

InApollo/*.md + optionally Themis/question_bank.md
OutIris/session – summary.md or Iris/exam_summary.md
HTMLAthena → STUDY PATH/summary.html via iris_build.py
Cmd/Iris
⚖
Themis
Working + Responsive

Oral examiner. Phase 1 (working): builds a Q&A reference bank from XLSX + Apollo notes, then auto-triggers Iris in Question-Aware mode. Phase 2 (responsive): live oral exam graded on two independent dimensions — source accuracy and knowledge depth.

InPh1: xlsx + Apollo/*.md  |  Ph2: question_bank.md
OutThemis/question_bank.md + Themis/sessions/ transcripts
HTMLAthena → question_bank.html + transcript – date.html
Cmd/Themis

Themis — Two-Phase Pipeline
Working Agent
Phase 1 — Preparation
1
Read Themis/*.xlsx — extract every question verbatim, preserving numbering
2
Find answers in priority order:
Apollo/*.md → ocr/*.md → Fortran source
3
Write Themis/question_bank.md — one entry per question with full answer + source attribution. Unanswered → Not found in study materials
4
Report gaps to student. Offer re-dispatch of Apollo for missing topics.
5
Auto-dispatch Iris in Question-Aware mode — always, no user prompt required
Responsive Agent
Phase 2 — Oral Exam
1
Read question_bank.md — refuse and abort if file does not exist
2
Ask a question (student picks topic or Themis selects)
3
Student answers (voice or text)
4a
Source grade — strict comparison against the question_bank.md answer field
4b
Knowledge grade — Themis's own live inference of what a complete answer requires
5
Both grades shown together. If they diverge, Themis notes the reason explicitly.
6
Loop or end — save graded transcript to Themis/sessions/YYYY-MM-DD.md
Iris — Two Modes
Student-triggered
Standard Mode
1
Read specified Apollo/*.md files
2
Produce 4-section exam-prep summary (~10–15% of source):
Key Concepts · Key Equations · Key Algorithms · Watch Out For
3
Write Iris/session – summary.md
4
GAIA dispatches Athena → STUDY PATH/summary.html
Not dispatched automatically per session — only on explicit student request.
Auto — after Themis Phase 1
Question-Aware Mode
1
Read question_bank.md — build internal topic index from every question
2
Read all Apollo/*.md for content
3
Same 4 sections as Standard Mode
4
Add inline exam callouts for matched items:
EXAM QUESTION — Q3, Q7: "Describe the cooling instability criterion."
5
Add Coverage Map table — every Q# mapped to where it appears in the summary; gaps flagged explicitly
6
Write Iris/exam_summary.md — included in next summary.html rebuild
